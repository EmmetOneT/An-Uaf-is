DevLog

Week 1

Fixed enemy dead body following the player by changing their follow parameters to themselves when in the ‘dead state’. Set up random spawn for item pickups. Using ammo boxes, they can now spawn randomly in locations set by the developer. Using GIMP, I downscaled an image of a letter to be used in game to match the PSX aesthetic. In Blender, created plane and imported letter image as texture. Using a canvas, panel and textmeshpro and creating a letter prefab that can be edited so, the prefab can be brought into the scene where, text can be added after for ease of use. All players controls, aside from the ‘Use’ key get locked when reading so, when the player presses ‘Use’ while reading, they can stop interacting with the letter. Worked on ammunition pickup. Had some issues getting the Game manager to send over to the ammunition pickup. Player is set up to count ammunition so, the gun cant be fired without ammunition in the player inventory Continued trying to fix the ammunition pickup. Moved onto opening and closing doors with the raycast. Door plays animation and always opens away from player for ease of use in-game. Added a key that holds a string to be used with a unique door. This uses a compare string, comparing the doors ID with the Keys ID. Keys with strings are added to an array which gets looped when trying to open a locked door. Added another enemy type with 2 variations. This new enemy is a more traditional zombie type. They have a walk, attack, death and hit animation attached. They follow the player character. I went back to try to figure out what went wrong with my ammo pickup. I found out that it was the game manager. I was using the local variable where, it should have been a global so items like the pistol, player and letter could interact with it. Game now checks whether the player has the gun equipped. If not equipped and empty, the gun will make an empty click sound and not fire. If Player picks up the ammo, amount of bullets specified will be added to the gun, allowing the player to fire that amount of time before the gun goes back to empty. Youtube Video Link to Gun Fire: https://youtu.be/YAt3oMUxT6U Set up zombie spawner. Unlike the last enemy, this zombie now spawns zombies in spawn points that I have set in the testscene. Created a template to spawn in enemies on a trigger. Set up for ‘creeping’ game objects, where an item in the world will follow the player when not looking at it.
